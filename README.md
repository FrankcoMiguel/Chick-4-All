# Chick-4-All Restaurant Management System\n\nA modern full-stack restaurant management application built with **Spring Boot**, **MongoDB**, and **React**.\n\n## Architecture\n\n- **Backend**: Spring Boot 2.7.18 with MongoDB\n- **Frontend**: React 18 with Bootstrap 5\n- **Database**: MongoDB\n- **Security**: Spring Security with BCrypt password encoding\n\n## Features\n\n- **Dashboard**: Real-time order statistics and recent orders\n- **Customer Management**: Add, view, and search customers\n- **Order Management**: Track orders with status updates\n- **Menu Items**: Manage restaurant menu with categories\n- **User Authentication**: Secure login system\n\n## Prerequisites\n\n- Java 17+\n- Node.js 18+\n- MongoDB 4.4+\n- Maven 3.6+\n\n## Setup Instructions\n\n### 1. Clone and Navigate\n```bash\ncd /path/to/Chick4All\n```\n\n### 2. Setup MongoDB\n\n**Option A: Local MongoDB**\n```bash\n# Install MongoDB (macOS with Homebrew)\nbrew install mongodb-community\nbrew services start mongodb-community\n\n# Create database\nmongosh\nuse chick4all\n```\n\n**Option B: MongoDB Atlas (Cloud)**\n1. Create account at [MongoDB Atlas](https://www.mongodb.com/atlas)\n2. Create a cluster and get connection string\n3. Update `src/main/resources/application.properties`:\n```properties\nspring.data.mongodb.uri=mongodb+srv://username:password@cluster.mongodb.net/chick4all\n```\n\n### 3. Build and Run Backend\n\n```bash\n# Build the project\nmvn clean install\n\n# Run the Spring Boot application\nmvn spring-boot:run\n```\n\nThe backend will start on http://localhost:8080\n\n### 4. Setup and Run Frontend\n\n```bash\n# Navigate to frontend directory\ncd src/main/frontend\n\n# Install dependencies\nnpm install\n\n# Start React development server\nnpm start\n```\n\nThe frontend will start on http://localhost:3000\n\n### 5. Create Initial User\n\nCreate a user via API or add directly to MongoDB:\n\n```bash\ncurl -X POST http://localhost:8080/api/users \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"username\": \"admin\",\n    \"password\": \"password123\",\n    \"phone\": \"+1234567890\"\n  }'\n```\n\n## API Endpoints\n\n### Authentication\n- `POST /api/users/login` - User login\n- `POST /api/users` - User registration\n\n### Users\n- `GET /api/users` - Get all users\n- `GET /api/users/{id}` - Get user by ID\n- `PUT /api/users/{id}` - Update user\n- `DELETE /api/users/{id}` - Delete user\n\n### Customers\n- `GET /api/customers` - Get all customers\n- `POST /api/customers` - Create customer\n- `GET /api/customers/{id}` - Get customer by ID\n- `PUT /api/customers/{id}` - Update customer\n- `DELETE /api/customers/{id}` - Delete customer\n- `GET /api/customers/search` - Search customers\n\n### Items\n- `GET /api/items` - Get all items\n- `POST /api/items` - Create item\n- `GET /api/items/{id}` - Get item by ID\n- `PUT /api/items/{id}` - Update item\n- `DELETE /api/items/{id}` - Delete item\n- `GET /api/items/available` - Get available items\n- `GET /api/items/category/{category}` - Get items by category\n\n### Orders\n- `GET /api/orders` - Get all orders\n- `POST /api/orders` - Create order\n- `GET /api/orders/{id}` - Get order by ID\n- `PUT /api/orders/{id}` - Update order\n- `DELETE /api/orders/{id}` - Delete order\n- `GET /api/orders/customer/{customerId}` - Get orders by customer\n- `GET /api/orders/status/{status}` - Get orders by status\n- `PUT /api/orders/{id}/status` - Update order status\n- `GET /api/orders/stats/status-counts` - Get order statistics\n\n## Configuration\n\n### Application Properties\n\nKey configuration in `src/main/resources/application.properties`:\n\n```properties\n# Database\nspring.data.mongodb.host=localhost\nspring.data.mongodb.port=27017\nspring.data.mongodb.database=chick4all\n\n# Server\nserver.port=8080\n\n# Security\napp.jwtSecret=chick4allSecretKey\napp.jwtExpirationMs=86400000\n```\n\n## Development\n\n### Running in Development Mode\n\n1. **Backend**: Uses Spring Boot DevTools for hot reload\n2. **Frontend**: React development server with hot reload\n3. **Database**: MongoDB running locally or in cloud\n\n### Building for Production\n\n```bash\n# Build complete application (includes React build)\nmvn clean package\n\n# Run production JAR\njava -jar target/Chick-4-All-1.0-SNAPSHOT.jar\n```\n\nThe production build serves React from Spring Boot on http://localhost:8080\n\n## Sample Data\n\nTo add sample data, you can use the provided API endpoints or import directly to MongoDB.\n\n### Sample Customer\n```json\n{\n  \"name\": \"John Doe\",\n  \"birthDate\": \"1990-01-01\",\n  \"address\": {\n    \"street\": \"123 Main St\",\n    \"city\": \"Anytown\",\n    \"state\": \"CA\",\n    \"zipCode\": \"12345\",\n    \"country\": \"USA\"\n  },\n  \"platform\": \"MOBILE_APP\"\n}\n```\n\n### Sample Item\n```json\n{\n  \"name\": \"Fried Chicken Combo\",\n  \"description\": \"Crispy fried chicken with fries and drink\",\n  \"price\": 12.99,\n  \"category\": \"Chicken\",\n  \"available\": true\n}\n```\n\n## Troubleshooting\n\n### Common Issues\n\n1. **MongoDB Connection Issues**\n   - Check MongoDB is running: `brew services list | grep mongodb`\n   - Verify connection string in application.properties\n\n2. **Frontend Not Loading**\n   - Ensure Node.js and npm are installed\n   - Clear npm cache: `npm cache clean --force`\n   - Delete node_modules and reinstall: `rm -rf node_modules && npm install`\n\n3. **CORS Issues**\n   - Check SecurityConfig.java CORS configuration\n   - Verify frontend proxy in package.json\n\n4. **Build Issues**\n   - Ensure Java 17+ is installed and JAVA_HOME is set\n   - Clear Maven cache: `mvn dependency:purge-local-repository`\n\n## License\n\nThis project is licensed under the MIT License.\n\n## Contributing\n\n1.
